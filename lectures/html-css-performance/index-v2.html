Steps of a webpage being displayed?

  1. User type URL into the browser address bar
  2. The browser will lookup the hostname in DNS(Domain Name System), if it is not already in the browser cache
  3. The browser initiates a TCP connection with the server
  4. The browser sends an HTTP request to the web server
  5. The server handles the request and sends back a response(HTML file)
  6. The browser receives the response and starts parsing the HTML file
  7. The browser renders the HTML content
  8. The browser sends additional requests for objects embedded in the html(CSS files, images, javascript etc.)

Fonts optimization

Be aware of the text rendering delay

  • Check font usage
  • Use porgressive font format: WOFF2.0, WOFF, EOT
  • Use optimized font formats for each browser: Latin, Cyrillique

Preload your Webfont resources (the text rendering delay)




                        

@font-face {
font-family: 'Awesome Font';
font-style: normal;
font-weight: 400;
src: local('Awesome Font'),
   url('/fonts/awesome-l.woff2') format('woff2'), /* will be preloaded */
   url('/fonts/awesome-l.woff') format('woff'),
   url('/fonts/awesome-l.ttf') format('truetype'),
   url('/fonts/awesome-l.eot') format('embedded-opentype');
    unicode-range: U+000-5FF; /* Latin glyphs */
}

Be Wary of Expensive Properties

  • border-radius
  • box-shadow
  • transform
  • filter

What forces reflow

  • • Script manipulating the DOM(element addition, deletion, altering, or changing element order)
  • • Changing the ".class" attribute
  • • Adding, deleting and updating CSS styles
  • • CSS3 animation and transition property
  • • Content changes, such as a user typing text in an input box
  • • Changing the font
  • • Scrolling, resize window
  • • CSS pseudo classes activation :hover etc.

What forces repaint

  • Changing:
  • • color
  • • background
  • • border-color
  • • opacity
  • • visibility

WHAT create layers

  • Elements video, canvas, iframe
  • 3D-transformation: translate3d, translateZ
  • CSS filters
  • position: fixed
  • sibling with a lower z-index
  • etc.

Animation performance

Animating the dimensions or position of an element can cause the whole page to re-layout on every frame

Use GPU accelaration


#a, #b {
    position: absolute;
}

#a {
    left: 10px;
    top: 10px;
    z-index: 2;
    animation: move 1s linear;
}

#b {
    left: 50px;
    top: 50px;
    z-index: 1;
}

                    

// bad
@keyframes move {
    from { left: 30px; }
    to { left: 100px; }
}
                    

// good
@keyframes move {
    from { transform: translateX(0); }
    to { transform: translateX(70px); }
}
left use transform, right use left, width

Keep the objects being animated as high as possible at z-index. Ideally, right inside body
Remove complicated animation from static layer: position: absolute, position: fixed

Promote elements that you plan to animate

will-change

.element {
    // browsers that don't support will-change
    transform: translate3d(0, 0, 0);
    transform: translateY(0);
}
                        

.element {
    /* ... */
    transition: transform 1s ease-out;
}
.element:hover {
    will-change: transform;
}
.element:active {
    transform: rotateY(180deg);
}

Don't create lots of extra layers!!!!


// don't do it!!!!!!!!!!
* {
will-change: all;
}

Don't forget to remove will-change


var el = document.getElementById('element');
el.addEventListener('mouseenter', hintBrowser);
el.addEventListener('animationEnd', removeHint);
function hintBrowser() {
this.style.willChange = 'transform, opacity';
}
function removeHint() {
this.style.willChange = 'auto';
}
